InformationFlowDefinition
*************************

**Fully qualified class name**: DDICDIModels::DDICDILibrary::Classes::Process:::index:`InformationFlowDefinition`


Definition 
============ 
The relationships between parameters across steps is described by the information flow definition. Each relationship creates a pathway for an information object to follow.  

Examples 
========== 
The output of one parameter might be the input of another one. Think of a Business Process Model and :ref:`DDICDIModels::DDICDILibrary::Classes::DataDescription::Notation` (BPMN) diagram or an extract, transform, and load (ETL) pipeline definition as the sum of these relationships.  

Explanatory notes 
=================== 
In DDI Lifecycle an information flow definition was referred to in terms of a "binding".
